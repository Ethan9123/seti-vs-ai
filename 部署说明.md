# SETI 对战AI — 上线给朋友测试（部署说明）

这个 `seti_web/` 文件夹就是**完整可上线的网站**（5.7 MB，纯静态，无需服务器/数据库/编译）。
里面已包含全部可玩内容：137 基础卡 + 45 张 Space Agencies 扩展卡 + 信号令牌 + 快速开始 + 8 外星种族（官方 ET 卡）+ 22 组织 + 官方 Rival + 2-4 人多人 + AlphaZero AI + 存档/悔棋 + 攻略 + 手机自适应。

直接把**整个 `seti_web/` 文件夹的内容**传到你的网站即可，无需改动。打开后玩到的就是最新版。

---

## 方案一：你的 try-board-game.uk 已有主机（最快，不用 GitHub）
如果 try-board-game.uk 已经托管在某处（cPanel / 宝塔 / Netlify / FTP 等）：
1. 在网站根目录新建一个子目录，例如 `seti/`。
2. 把 `seti_web/` 里**全部文件**（`index.html`、7 个 `.js`、整个 `assets/`）上传到 `seti/`。
3. 访问 `https://try-board-game.uk/seti/` 即可游玩，把这个链接发给朋友。

> 想用子域名 `seti.try-board-game.uk` 也行：在主机控制台建子域名指向这个目录，再上传。

## 方案二：用 GitHub（免费托管 + 以后好更新，推荐当你没有现成主机时）
1. 在 GitHub 新建一个仓库（public），把 `seti_web/` 里的**内容**（不是文件夹本身）上传/推送到仓库根。
2. 仓库 **Settings → Pages → Build and deployment → Deploy from a branch → main / (root)**，保存。
3. 几分钟后得到 `https://你的用户名.github.io/仓库名/`，可直接玩。
4. 想绑定 `seti.try-board-game.uk`：仓库根放一个 `CNAME` 文件（内容写 `seti.try-board-game.uk`），再去域名 DNS 加一条 CNAME 记录指向 `你的用户名.github.io`。
   - GitHub Pages 自动签发 HTTPS。

> Cloudflare Pages / Netlify 也一样：连上这个 GitHub 仓库（或直接拖拽 `seti_web/` 文件夹），构建命令留空、输出目录就是根，再绑定自定义域名。

## 注意事项（已为你处理 / 需留意）
- **HTML 已改名为 `index.html`**：原文件名含中文 `对战AI`，做网址会变成丑陋的 %编码；用 index.html 后网址就是干净的目录地址。
- **纯相对路径、无任何外链/CDN**：已验证可在子目录下运行（`/seti/` 下全部资源 200，无 404）。
- **大小写敏感**：Linux 主机（含 GitHub Pages）区分大小写，本项目资源全部小写、引用一致，正常即可。
- **必须整套一起传**：`index.html` + 7 个 `.js`（cards_data / sa_cards / quickstart_cards / ai_nn / ai_weights / az_nn / az_weights）+ `assets/`，缺一不可。
- **不要上传**项目里其它文件（训练脚本 `_*.js`/`az_*`/`*.py`、PDF 规则书、`*.md`、`.backup` 等）——它们与游戏无关，且体积巨大（~140MB）。本 `seti_web/` 已只挑出需要的 5.7MB。
- 朋友手机也能玩（已做移动端自适应）；存档用浏览器 localStorage，换设备/清缓存会丢档。
